## pqc_ang_v1.py 

1. 数据集包含336000个样本，每个样本有256个特征，即每个样本是一个256维向量。
2. 选取固定的8个特征作为输入，其他特征作为输出。
3. 模型结构如下：
- 归一化：通常RSRP值范围较大，对输入特征进行归一化处理，使其在0到1之间。
- 使用角度编码：使用8个量子比特，将输入特征编码为角度，方便计算。
- 量子电路：使用8量子比特Strongly Entangling Layers,堆叠L=3层
- 量子测量：计算8个量子比特在Pauli-Z基下的测量期望值，得到8维输出。
- 经典回归：使用小的MLP，8维到64维，再到256维，作为经典回归模型，将量子输出映射到256维的预测结果。
4. 损失函数：使用均方误差 (MSE) 
5. 优化器：使用Adam优化器，学习率为0.001。
6. 训练过程：迭代训练模型，直到损失收敛或达到预设的训练轮数。


## pqc_reup_v1.py

1. 数据集包含336000个样本，每个样本有256个特征，即每个样本是一个256维向量。
2. 选取256个特征中等距固定的48个特征作为输入，其余特征作为预测输出。
3. 模型结构如下：
- 使用data re-upload技术，将输入特征分成12个块，每个块是4个特征，每个量子比特编码编码4个特征逐块输入量子电路进行处理。例如一个块里的4个特征是x1,x2,x3,x4，组成4维列向量。取一个3*4矩阵，将4个特征映射到3维空间，再取一个3维向量
作为偏置，得到一个3维列向量。将这个3维列向量作为输入，使用U3门编码到一个量子比特上。所以每个块编码进一个量子比特，12个块编码进12个量子比特。
- 量子电路：使用12量子比特Strongly Entangling Layers,堆叠L=3层
- 量子测量：计算12个量子比特在Pauli-Z基下的测量期望值，得到12维输出。
- 经典回归：使用小的MLP，12维到64维，再到256维，作为经典回归模型，将量子输出映射到256维的预测结果。
4. 损失函数：使用均方误差 (MSE)
5. 优化器：使用Adam优化器，学习率为0.001。
6. 训练过程：迭代训练模型，直到损失收敛或达到预设的训练轮数。
7. 评估指标：使用均方误差 (MSE) 和平均绝对误差 (MAE) 评估模型性能。
8. 结果分析：训练完成后，分析模型的预测结果与真实值之间的差异，绘制预测结果与真实值的对比图，计算MSE和MAE指标，评估模型的预测性能。
9. 数据集：/Users/luxian/DataSpace/beam_pre/sls_beam_data_spatial_domain_vivo.mat
10. 训练过程和结果分析保存在本仓库内的 pqc_reup_v1_output 目录


pqc_reup_v1.py 输出文件命名规则
1. 模型参数文件
命名格式: model_params_epoch_{epoch数}.pth
示例: model_params_epoch_5.pth
用途: 保存训练完成后的模型参数
2. 训练过程数据
命名格式: training_data_epoch_{epoch数}.json
示例: training_data_epoch_5.json
用途: 保存训练过程中的损失和MAE数据
3. 评估结果
命名格式: evaluation_results_epoch_{epoch数}.json
示例: evaluation_results_epoch_5.json
用途: 保存模型评估的预测结果和指标
4. 时间戳模型文件
命名格式: model_{时间戳}.pth
示例: model_20260209_154022.pth
用途: 带时间戳的模型备份
5. 评估指标文件
命名格式: evaluation_metrics.json
用途: 保存基本的评估指标
6. 配置文档
命名格式: config_{时间戳}.md
示例: config_20260209_154022.md
用途: 保存实验配置和结果
7. 继续训练专用文件
在继续训练版本中，还会生成：
训练数据: training_data_epoch_{累计epoch数}.json
输出目录
所有文件都保存在：pqc_reup_v1_output/ 目录中

文件命名特点
使用 epoch 数字来标识训练轮次
时间戳格式为 YYYYMMDD_HHMMSS
JSON文件用于存储结构化数据
PTH文件用于存储PyTorch模型参数
